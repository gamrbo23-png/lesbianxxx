Part 1 — Create New Project
01
Open ISE Design Suite
Launch ISE Project Navigator from the desktop or Start Menu. Wait for the splash screen to finish loading.
02
New Project Wizard
Go to File → New Project. The New Project Wizard will open.
03
Set Project Name & Location
Enter a Project Name (e.g. and_gate_lab). Set the Project Location to a folder on your PC. Select HDL as Top-level source type.
Location: C:\xilinx_projects\and_gate_lab
04
Device & Family Settings
Set the following properties for the FPGA target:

Family: Spartan6 (or your board)
Device: XC6SLX9
Package: TQG144
Speed Grade: -2
Simulator: ISim (VHDL/Verilog)
Preferred Language: Verilog
TIP: If you don't have a real board, any Spartan3/6 or Artix7 device works for simulation.
05
Finish Wizard
Click Next → review the summary → click Finish. The Project Navigator workspace will open.
Part 2 — Add Verilog Source
01
Add New Source File
In the Design panel (left side), right-click on the project name → select New Source.
02
Select Verilog Module
In the New Source Wizard, select Verilog Module. Enter the file name matching your module (e.g. and_gate.v). Click Next.
03
Define Port Names (Optional)
You can define ports here, or skip and write them manually. Click Next → Finish.
TIP: It's faster to skip port definition here and paste full code directly from this app's COPY button.
04
Paste Your Verilog Code
The file opens in the ISE editor. Delete the auto-generated template. Go back to the Repo tab in this app, hit COPY on any gate, then paste it into the ISE editor (Ctrl+V). Save with Ctrl+S.
Part 3 — Synthesize Design
01
Select Implementation View
In the Design panel, make sure the dropdown at the top is set to Implementation (not Simulation).
02
Run Synthesize — XST
In the Processes panel below the Design panel, double-click Synthesize — XST. A green tick ✓ means success. A red X means syntax errors in your code.
03
Check Console Output
The Console at the bottom shows synthesis messages. Look for: Process "Synthesize - XST" completed successfully
ERROR FIX: If you see "unexpected token" errors, check your Verilog code for missing semicolons, wrong module names, or mismatched begin/end.
Part 4 — Add Testbench & Simulate
01
Add New Source — Verilog Test Fixture
Right-click project → New Source → select Verilog Test Fixture. Name it tb_and_gate.v. Click Next.
02
Associate with Your Module
ISE will ask which module to associate the testbench with. Select your module (e.g. and_gate). Click Next → Finish.
03
Paste Testbench Code
Delete the auto-generated template. Go to the Benchmark tab in this app, hit COPY on the matching testbench (e.g. AND Gate TB), paste into the ISE editor. Save with Ctrl+S.
04
Switch to Simulation View
In the Design panel dropdown, change from Implementation to Simulation. Your testbench file should now appear as the top module.
TIP: If the testbench doesn't appear, right-click it in the file list → Set as Top Module.
05
Run iSim Behavioral Simulation
In the Processes panel, expand iSim Simulator → double-click Simulate Behavioral Model. iSim window will open automatically.
06
View Waveform in iSim
In the iSim window:

• Click Run All button (▶▶) to run the full simulation
• Or type run 100ns in the console
• The waveform panel shows signal values over time
• Zoom with Ctrl+Scroll or the zoom toolbar
TIP: Click the Zoom to Full View button (magnifier icon) to see all signals at once.
07
Verify Output
Check waveform values match the truth table. For AND Gate: output y should be 1 only when both a=1 AND b=1. All other combinations should output 0.
Part 5 — Common Errors & Fixes
!
ERROR: Module not found
Cause: Testbench module name doesn't match the design module name.
Fix: Check that and_gate uut(...) in your TB matches the module name exactly in your source file.
!
ERROR: Unexpected token / Syntax Error
Cause: Incorrect Verilog syntax — missing ;, wrong keywords.
Fix: Use the COPY button from this app — the code is verified and correct.
!
iSim Shows Flat Line / No Signal
Cause: Simulation ran but signals aren't added to the waveform.
Fix: In iSim, go to Default.wcfg panel → right-click signals → Add to Waveform. Then click Run All again.
!
ERROR: File not in project
Cause: Verilog file saved outside project folder.
Fix: Always save files inside your project directory. Or use Project → Add Source to manually add an existing file.
!
Synthesis Passes but Simulation Fails
Cause: Testbench not set as top module in Simulation view.
Fix: In Design panel → Simulation view → right-click the TB file → Set as Top Module.

Part 7 — Processes Explained
Process NameWhat It Does
Synthesize — XSTConverts Verilog code to gate-level netlist. First step after writing code.
Implement DesignMaps netlist to FPGA resources (for real hardware). Not needed for simulation.
Generate Prog. FileCreates .bit file to flash onto FPGA board.
Simulate BehavioralRuns testbench before synthesis — fastest, uses your Verilog directly.
Simulate Post-SynthRuns simulation after synthesis — checks synthesized netlist.
View RTL SchematicShows auto-generated circuit diagram of your design.
Check SyntaxQuick syntax check without full synthesis. Use this first.

S=0 R=0 @ ↑clk → q = 0 (hold)
